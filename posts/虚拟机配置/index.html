虚拟机配置

Posted on Sep 17, 2023

个人常用虚拟机配置，持续更新。。。

zsh

下载

sudo apt install zsh

chsh -s /bin/zsh

插件

sh -c "$(curl -fsSL https://raw.github.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh}/plugins/zsh-syntax-highlighting
git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh}/plugins/zsh-autosuggestions

修改zshrc

plugins=(
	git 
	zsh-syntax-highlighting 
	zsh-autosuggestions
)

主题

git clone --depth=1 https://github.com/romkatv/powerlevel10k.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/themes/powerlevel10k

# 修改zshrc
ZSH_THEME=powerlevel10k/powerlevel10k

字体

下载，解压，将 ttf 文件放入 一下文件夹

# 给用户自己用
mkdir -p ~/.fonts

# 全部用户
/usr/share/fonts

执行命令

sudo mkfontscale
sudo mkfontdir
sudo fc-cache

网络

  • 出现获得不了IP地址的情况
  1. kali 修改 /etc/network/interfaces
# 找网卡名
ifconfig

# 修改 interfaces 文件
auto eth0
iface eth0 inet dhcp

# 重启服务
/etc/init.d/networking restart
ifconfig eth0 down    
ifconfig eth0 up
  1. ubuntu 使用命令，但是重启后又不行
  • ifconfig 配置IP
ifconfig ens33 <IP> netmask <mask>
route add default gw <gateway>
  • 然后使用如下命令
sudo /sbin/dhclient
  1. ubuntu 修改 文件
sudo vim /etc/netplan/01-network-manager-all.yaml

# 修改内容
network:
  version: 2
  renderer: NetworkManager
  ethernets:
     eth0:
       dhcp4: yes
       addresses: []

# 执行命令
sudo netplan try
sudo netplan apply
  • 在修改 /etc/NetworkManager/NetworkManager.conf 配置文件的 managed = true
  • reboot
  • 后来看了这个文章解决 Ubuntu20.04不能连接网络及解决开机自动获取 ip 问题
  1. 如下命令
sudo nmcli networking off 
sudo nmcli networking on

sudo 免密

修改 /etc/sudoers 添加用户或者组

# 指定命令免密
<name> ALL=(ALL:ALL) NOPASSWD:/bin/useradd,/bin/chown

# 所有的都免密
<name> ALL=(ALL:ALL) NOPASSWD:ALL

内存问题

压缩

删除快照后任显示存在快照，如何删除干净：新建一个，然后删除，VM会把之前的快照合并后删除。

  • 删除所有快照和整合快照功能常见问题解答(vmware.com)

当我们删除部分大文件时，发现自己磁盘的内存没有增加。vmdk文件不会主动回缩，需要手动做shrink。并且不能有快照

$ sudo vmware-toolbox-cmd disk list
$ sudo vmware-toolbox-cmd disk shrink /

然后重启

kali 无法进入console

  • 我的情况是 fstab 出现问题，共享文件夹修改 fstab
  1. 重启，进入 grub加载项
  2. e 进入配置
  3. 修改 linux 中的 ro => rw 并加入一句 init=/bin/bash
  4. f10 进入系统
  5. 修改 fstab，重启重新获得界面
