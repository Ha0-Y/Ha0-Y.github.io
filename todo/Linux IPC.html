InterProcess Communication 进程间通信

process

进程：正在进行的一个过程或者说一个任务。而负责执行任务则是cpu。

  • 在Linux下使用fork() 可以创建一个进程
  • 退出：可以使用kill 或者 exit 退出进程
  • wait：因为子进程需要被父进程回收，wait避免父进程结束但是子进程没有结束而变为僵尸进程
    • wait 函数其实是等待子进程的信号，等待接收到终止或者停止信号。
    • 终止信号：SIGTERM，代表可以回收。
    • 停止信号：进程收到SIGSTOP信号后，会进入T (TASK_STOPPED or TASK_TRACED)，挂起状态或跟踪状态。如果要使得被挂起的进程重新运行，使用SIGCONT信号触发。还可以使用 ptrace 使其继续运行。
  • 获得pid processer identifer
  • kill: 向进程发送一个信号，只不过默认是 SIGKILL。也可以唤起 SIGSTOP 的进程

标准

当我们在 Linux 系统中进行进程间通信时，例如信号量，消息队列，共享内存等方式，会发现有System V以及POSIX两种类型。

System V 以及POSIX 对信号量、共享内存、消息队列等进程之间共享方式提供了自己的解决方案。

  • UNIX两大贡献者贝尔实验室和BSD，在进程之间通信侧重不同，前者基于内核对进程之间的通信手段进行了改进，形成了System V IPC，而后者则是基于网络形成了套接字。
  • POSIX(Portable Operating System Interface for Computing Systems)是由IEEE 和ISO/IEC 开发的一簇标准，目的是为运行在不同操作系统上的软件提供统一的接口，实现者则是不同的操作系统内核开发人员。

POSIX 在无竞争条件下，不需要陷入内核，其实现是非常轻量级的; System V 则不同，无论有无竞争都要执行系统调用，因此性能落了下风。

POSIX 比较看重 名称，所以可以看出在执行相应的 open 函数时时中都会存在 name 这个参数
System V 创建时则是 key_t 关联。这个值由用户定义
