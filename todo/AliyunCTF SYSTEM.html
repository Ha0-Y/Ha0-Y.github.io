MDL

memory descriptor list

跨一系列连续虚拟内存地址的 I/O 缓冲区可以分布在多个物理页中，并且这些页面可以是不连续的。 操作系统使用 内存描述符列表 (MDL) 来描述虚拟内存缓冲区的物理页面布局。

API

MmAllocateContiguousMemory：分配一系列连续的非分页物理内存，并将其映射到系统地址空间。MmAllocateContiguousMemory分配的内存未初始化

释放由 MmAllocateContiguousMemoryXxx 例程分配的一系列物理连续内存。

给定缓冲区的起始地址和长度， IoAllocateMdl 分配内存描述符列表 (MDL) 足以映射缓冲区。

创建一个MDL结构体，而这个结构体描述给出的VurtualAddress

  • StartVa：page开始的地址
  • ByteOffset：在page内的偏移
  • ByteCount: 大小

IoFreeMdl 释放调用方分配的内存描述符列表 (MDL) 。

MmBuildMdlForNonPagedPool 接收指定非分页虚拟内存缓冲区的 MDL，并更新它以描述基础物理页。

MmMapLockedPagesSpecifyCache 将 MDL 描述的物理页面映射到虚拟地址，并使调用方能够指定用于创建映射的缓存属性。

AccessMode：KernelMode 或 UserMode。 几乎所有驱动程序都应使用 KernelMode。

MmNonCached：请求的内存不应由处理器缓存。

Exploit

MmMapLockedPagesSpecifyCache 地址映射 AccessMode 为 UserMode

翻了几篇Windows的漏洞利用，寻找下这个函数相关：用户态可以访问内核内存，使用VirtualAlloc可以获得内存。
